知识点

History Mode Hash vs HTML5

在创建路由器实例时，history 配置允许我们在不同的历史模式中进行选择。

Hash Mode

hash 模式由 createWebHashHistory() 创建：

它在内部传递的实际 URL 之前使用了一个哈希字符（#）, #以及#后面的字符称之为hash，用window.location.hash 读取。

特点：hash虽然在URL中，但不被包括在HTTP请求中；用来指导浏览器动作，对服务端安全无用，hash不会重加载页面。由于这部分 URL 从未被发送到服务器，所以它不需要在服务器层面上进行任何特殊处理。

缺点： 对SEO有不好的影响。

The purpose of Hash Mode is to imitate a file path without having to setup any special server configurations. Allows us to not have to setup our server to handle single entry point.

HTML5 模式

用 createWebHistory() 创建 HTML5 模式，推荐使用这个模式：

当使用这种历史模式时，URL 会看起来很 “正常”，例如 https://example.com/user/id

由于 vue 创建的应用 是一个单页的客户端应用，如果没有适当的服务器配置，用户在浏览器中直接访问 https://example.com/user/id，就会得到一个 404 错误。

要解决这个问题，需要做的就是在你的服务器上添加一个简单的回退路由（simple catch-all fallback route ）。如果 URL 不匹配任何静态资源，它应提供与你的应用程序中的 index.html 相同的页面。
